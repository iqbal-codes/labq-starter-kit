# Product

## Register

product

## Users

Mixed audience: technical builders (developers, agencies, technical founders building or demoing internal tools) and day-to-day ops users (non-technical team members managing contacts, inventory, and deals in an operational context). The builder audience evaluates architecture and extensibility; the ops audience needs fast, clear workflows for daily data management.

## Product Purpose

A modular internal tools platform (Odoo-lite inspired) that serves as a foundation for building CRM, Inventory, and future business modules. Exists to prove that internal tools can have consistent UI, strong architecture, organization-level module activation, and a modern TypeScript stack — without the weight of a full ERP. Success is a 5–10 minute demo flow that feels smooth, professional, and extensible.

## Brand Personality

Clean, capable, calm. The interface should feel like a well-organized workspace: everything in its place, nothing shouting for attention, fast to navigate. Confidence comes from precision and restraint, not from visual excess.

## Anti-references

- **SaaS dashboard clichés**: Hero-metric cards with big numbers, gradient text, generic icon grids, purple-blue gradients everywhere, "01 / 02 / 03" numbered eyebrows on every section. The template that looks like every other AI-generated dashboard.
- **Enterprise/Outlook style**: Heavy chrome, dense toolbars, gray-on-gray everywhere, Windows-style UI, cluttered sidebars with tiny icons, dialog-heavy workflows.
- **Over-designed / flashy**: Glassmorphism as default, neon accents, excessive animations, trend-chasing visual effects, gradient text, decorative blurs.

## Design Principles

1. **Clarity over decoration** — Every element earns its place through function. If it doesn't help the user complete a task or understand their data, it doesn't belong.
2. **Speed as UX** — Fast interactions, immediate feedback, no unnecessary loading states. The interface should feel responsive at every layer.
3. **Modular by nature** — Components and features compose, not duplicate. The system is built to extend, and the UI reflects that — consistent patterns across modules, shared vocabulary.
4. **Data density without chaos** — Show what matters, organize it well. Tables, lists, and forms should feel information-rich but never cluttered. Hierarchy comes from spacing and typography, not from visual noise.
5. **Calm confidence** — No shouting, no noise. The interface is assured through restraint: tight layout, precise typography, minimal ornamentation. Confidence is felt, not performed.

## Accessibility & Inclusion

WCAG AA standard: 4.5:1 minimum contrast for body text, 3:1 for large text. Full keyboard navigation and screen reader support. Reduced motion support via `prefers-reduced-motion`. No interactions gated on color alone. Focus states visible and consistent.
